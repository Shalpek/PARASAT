Начать лучше не с кода, а с **минимального ТЗ и структуры проекта**. У тебя компания занимается профессиональной химией и расходными материалами для HoReCa: гостиниц, ресторанов, кафе и кейтеринга . Поэтому сайт должен выглядеть не как обычный лендинг, а как **каталог-магазин с заявками**.

## 1. Сначала определить цель проекта

Твоя цель для практики:

> Разработать веб-сайт интернет-магазина для компании Parasat Product Astana, позволяющий клиентам просматривать каталог товаров, добавлять товары в корзину и отправлять заявку на заказ, а администратору — управлять товарами и заявками.

То есть проект должен включать:

* главную страницу;
* каталог товаров;
* карточку товара;
* корзину;
* форму заявки;
* админ-панель;
* базу данных;
* адаптивный дизайн.

## 2. Лучший план действий

### Этап 1. Подготовка проекта

Сначала нужно создать структуру проекта:

```text
parasat-shop/
│
├── frontend/
├── backend/
├── database/
├── docs/
└── README.md
```

В папку `docs` можно потом добавить:

```text
ТЗ проекта
Архитектура
Описание базы данных
Скриншоты сайта
Отчет по практике
```

## 3. Выбрать стек технологий

Для практики я бы выбрал такой вариант:

```text
Frontend: React + TypeScript + Tailwind CSS
Backend/Database: Firebase
Admin panel: React
```

Для практики самый удобный вариант — **React + Firebase**, потому что можно быстрее сделать авторизацию, базу данных и хранение товаров.

## 4. Сначала сделать MVP

Не надо сразу делать идеальный маркетплейс. Сначала делаем минимальную рабочую версию.

### MVP сайта должен включать:

1. Главную страницу.
2. Каталог товаров.
3. Фильтрацию по категориям.
4. Страницу товара.
5. Корзину.
6. Форму оформления заявки.
7. Сохранение заявки в базу данных.
8. Админ-панель.
9. Добавление, редактирование и удаление товаров.
10. Просмотр заявок.

Это уже будет выглядеть как полноценный проект для практики.

## 5. Структура страниц

Сайт можно построить так:

```text
/                  Главная страница
/catalog           Каталог товаров
/product/:id       Страница товара
/cart              Корзина
/checkout          Оформление заявки
/about             О компании
/contacts          Контакты
/admin/login       Вход администратора
/admin             Панель администратора
/admin/products    Управление товарами
/admin/orders      Управление заявками
```

## 6. Категории товаров

По материалам компании можно сделать такие категории:

```text
1. Кухонная химия
2. Средства для уборки
3. Индустриальная химия
4. Дезинфицирующие средства
5. Прачечные средства
6. Личная гигиена
7. Расходные материалы
8. Одноразовая посуда
9. Бумажная продукция
10. Средства индивидуальной защиты
```

## 7. Что сделать в первую очередь

Я бы начала именно так:

### День 1 — ТЗ и структура

Сделать файл `README.md` и описать:

```text
Название проекта: Parasat Product Astana Shop

Цель: разработка интернет-магазина профессиональной химии и расходных материалов для HoReCa.

Пользователи:
- клиент;
- администратор.

Основные функции:
- просмотр товаров;
- поиск и фильтрация;
- корзина;
- заявка на заказ;
- управление товарами;
- управление заявками.
```

### День 2 — Дизайн страниц

Сначала нарисовать простую структуру:

```text
Header
Hero-блок
Категории
Популярные товары
О компании
Партнеры
Контакты
Footer
```

Можно пока без Figma, прямо в коде.

### День 3 — Frontend

Создать React-проект:

```bash
npm create vite@latest parasat-shop -- --template react-ts
cd parasat-shop
npm install
npm install react-router-dom axios
npm install -D tailwindcss
```

Потом создать страницы:

```text
HomePage.tsx
CatalogPage.tsx
ProductPage.tsx
CartPage.tsx
CheckoutPage.tsx
AdminPage.tsx
```

### День 4 — Данные товаров

Сначала можно сделать временный файл:

```text
products.ts
```

Туда добавить товары:

```ts
{
  id: 1,
  name: "Briller",
  category: "Кухонная химия",
  description: "Средство для удаления накипи",
  price: 0,
  image: "/products/briller.jpg"
}
```

Сначала цена может быть `Уточнить цену`, потому что у B2B-магазинов часто цена зависит от объема заказа.

### День 5 — Корзина

Сделать функции:

```text
Добавить в корзину
Удалить из корзины
Изменить количество
Посчитать количество товаров
```

### День 6 — Форма заявки

Форма должна содержать:

```text
Имя
Телефон
Город
Название компании
Комментарий
Список товаров из корзины
```

После отправки заявка сохраняется в базу.

### День 7 — Админ-панель

В админке сделать:

```text
Добавить товар
Редактировать товар
Удалить товар
Посмотреть заявки
Изменить статус заявки
```

Статусы заявки:

```text
Новая
В обработке
Выполнена
Отменена
```

Сайт предназначен для автоматизации процесса демонстрации товаров и приема заявок от клиентов. ```

```text
6. Сделать каталог.
7. Сделать карточку товара.
8. Сделать корзину.
9. Сделать форму заявки.
10. Подключить базу данных.
11. Сделать админ-панель.
12. Добавить адаптивность.
13. Сделать скриншоты.
14. Описать архитектуру.
15. Подготовить отчет по практике.
```
